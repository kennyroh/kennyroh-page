Kenny Roh

nextjs

“Nedt.js SSR, CSR, SSG, ISR 그리고 하이드레이션”

Hydration

사전적 의미는 “수분공급, 수분유지, 수화작용” 입니다. 화장품, 자동차 같은 곳에 사용되는 걸로 보입니다.
그리고 웹 용어 하이드레이션은 서버 사이드 렌더링(SSR) 또는 정적 사이트 생성(SSG)을 통해 생성된 HTML에 클라이언트 사이드 JavaScript가 추가되어 동적으로 웹 페이지가 변환됨을 말합니다.
이 과정을 통해 서버에서 렌더링된 정적 페이지가 클라이언트에서 동적인 React 애플리케이션으로 작동하게 됩니다.

서버 사이드 렌더링(SSR, Server Side Randering)

서버 사이드 렌더링은 서버에서 페이지를 렌더링하는 방식입니다. getServerSideProps 함수를 사용하여 SSR을 구현할 수 있습니다.

클라이언트 사이드 렌더링(CSR, Client Side Randering)

클라이언트 사이드 렌더링은 브라우저에서 페이지를 렌더링하는 방식입니다. React의 useState와 useEffect를 사용하여 CSR을 구현할 수 있습니다.

정적 사이트 생성(SSG, Static Site Generation)

정적 사이트 생성은 빌드 시점에 페이지를 렌더링하는 방식입니다. getStaticProps 함수를 사용하여 SSG를 구현할 수 있습니다.

ISR(ISR, Incremental Static Regeneration)

ISR은 필요에 따라 새로운 데이터를 렌더링하여 페이지를 업데이트하는 방식입니다. getStaticProps 함수와 revalidate 옵션을 활용하여 ISR을 구현할 수 있습니다.

Next.js는 다양한 렌더링 방식을 지원하여 개발자가 상황에 맞게 최적의 방식을 선택할 수 있게 해줍니다.

이런 선택지가 있구나라는 생각도 들면서
프론트앤드 개발자의 역량과 전문성이 더욱더 중요해질거라 생각합니다.
저는 찍먹만 하고 넘어가고 있습니다.
