Angular 고급 개발자가 되기 위해 알아두어야 할 주요 요소들

1. 정확한 타입 사용

가능한 한 ‘any’를 사용하는 것을 피하고, 타입을 사용하세요. Angular와 TypeScript를 사용하는 주요 이유 중 하나는 강력한 타입 시스템을 통한 안정성을 확보하기 위한 것입니다【11†source】.

interface User {
  id: number;
  name: string;
}

let user: User;
user = {
  id: 1,
  name: "John Doe"
};

2. package-lock.json 이해

package-lock.json은 앱이 사용하는 올바른 종속성 버전을 나열합니다. 정확한 모듈 버전을 얻으려면 npm ls module-name을 실행하세요【11†source】.

3. 개발 및 프로덕션 차이 인식

개발 모드와 프로덕션 빌드 사이에는 다른 작동 방식이 있을 수 있으므로, 개발 모드에서 모든 것이 정상적으로 작동한다고 해서 프로덕션 빌드에서도 동일하게 작동한다고 가정하지 마세요【12†source】.

4. 빌드 최적화와 vendor-chunk 설정 이해

빌드 최적화를 사용하면 라이브러리 코드가 번들에 포함되어 크기가 작아집니다. ‘vendorChunk’을 true로 설정하면 라이브러리가 별도의 vendor chunk에 위치하게 됩니다. 이 두 가지 설정은 서로 연관되어 있으므로, 어떤 시나리오에서 어떤 설정을 사용해야 하는지 이해하는 것이 중요합니다【13†source】.

5. 서비스 제공 방식 이해

서비스는 제공된 모듈에서 싱글톤입니다. 다른 모듈 제공자에게 추가하면 새 인스턴스가 생성됩니다【14†source】.

import { Injectable } from '@angular/core';
@Injectable({
  providedIn: 'root',
})
export class UserService {
}

6. Injectable에서 ‘providedIn’ 사용 고려
