Angular: Path 파라미터, Matrix URI 파라미터, Query 파라미터, Fragment.

Kenny Roh
2023-05-27
typescript

Angular에서는 Path 파라미터, Matrix URI 파라미터, Query 파라미터, 그리고 Fragment를 사용해 파러미터를 전달합니다.

Path 파라미터는 URL의 일부로서, 특정 라우트나 컴포넌트를 식별하는 데에 사용됩니다.
http://example.com/api/users/1에서 1이 Path 파라미터가 됩니다.

Matrix URI 파라미터는 세미콜론(;)으로 구분된 키-값 쌍의 형태로 제공됩니다.
http://example.com/api/users;id=1에서 id=1이 Matrix URI 파라미터가 됩니다.
Matrix URI의 창시자는 Tim Berners-Lee입니다​.

Query 파라미터는 URL의 끝부분에 위치하며, 주로 페이지네이션, 필터링, 검색 등의 기능에 사용됩니다.
http://example.com/api/users?id=1에서 id=1이 Query 파라미터가 됩니다.

Fragment는 URL의 맨 끝에 위치하며, 주로 웹 페이지 내의 특정 섹션을 가리키는 데에 사용됩니다.
http://example.com/api/users#section1에서 section1이 Fragment가 됩니다.

이런 것이 있구나 정도 알아두면 좋을 것 같습니다.
