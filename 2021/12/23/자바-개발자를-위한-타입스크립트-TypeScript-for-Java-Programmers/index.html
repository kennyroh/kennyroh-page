TypeScript for Java Programmers

React, Vue, Angular 이 3개중 1-2개월 살펴 보고 Angular의 길로 가기로 결정했습니다.

1. Co-learning JavaScript

TypeScript는 JavaScript의 상위 집합(superset)이므로, 먼저 JavaScript에 대한 이해를 갖는 것이 중요합니다.

2. OOP in TypeScript

TypeScript는 클래스, 인터페이스, 상속 등과 같은 Java와 유사한 OOP 개념이 있습니다.

3. Rethinking the Class

3.1 Nominal Reified Type Systems

Java는 nominal reified type system을 사용합니다. 이는 타입의 이름이 중요하다는 것을 의미합니다.
TypeScript는 이와 다르게 structural type system을 사용합니다.

3.2 Types as Sets

TypeScript에서 타입은 집합일수도 있습니다.

3.3 Erased Structural Types

TypeScript는 구조적 타입 시스템을 사용한다.

3.4 Empty Types

TypeScript에서 어떠한 값도 가질 수 없는 타입이 있습니다.

3.5 Identical Types

TypeScript에서는 두 타입이 구조적으로 동일하다면, 그 두 타입은 동일하다고 간주됩니다.

3.6 Reflection

TypeScript에서는 네이티브 리플렉션을 지원하지 않습니다.
