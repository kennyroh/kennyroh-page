Spring Framework의 의존성 주입

의존성 주입(Dependency Injection, DI) 프로그램을 더 간단히 다루고 코딩 방식입니다.
이것은 Spring Framework를 사용하는 이유이기도 합니다.

의존성 주입이란?

이 개념을 보여주는 간단한 예입니다:

이 예제에서, TextEditor 클래스는 내부에서 SpellChecker를 새로 만들고 있습니다.
SpellChecker 클래스를 new 로 생성하고 있습니다.
이것은 TextEditor 클래스 가 new SpellChecker() 에 의존한다고 볼수 있습니다.

이런 의존을 해결하기 위해 Dependency Injection을 사용합니다.

Spring에서의 의존성 주입

이제 Spring의 의존성 주입을 어떻게 사용할 수 있는지 코드를 봅니다:

SpellChecker class:

Spring의 @Autowired 태그를 사용하면, SpellChecker가 TextEditor가 만들어질 때 자동으로 전달됩니다(주입됩니다).
그래서, TextEditor는 SpellChecker를 new 로 생셩 하지 않아도 됩니다.
결합이 느슨해졌다고 표현합니다.

결론

의존성 주입은 클래스들을 덜 연결되게 만들고(느슨한 연결이 되도록 만들고),
간단히 다룰수 있습니다.
