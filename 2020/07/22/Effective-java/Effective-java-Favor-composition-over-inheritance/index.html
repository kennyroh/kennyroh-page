상속보다는 컴포지션을 선호하라

상속과 컴포지션은 클래스가 다른 클래스의 기능을 사용하는 두 가지 주요 방법입니다.
“이펙티브 자바” 책에서는 상속보다 컴포지션 선호를 강조합니다.

상속을 사용하는 경우,
Dog 클래스는 Animal 클래스를 상속받아 makeSound() 메서드를 오버라이드합니다.

상속은 클래스 간의 강력한 결합을 만들며, 부모 클래스의 변경이 자식 클래스에 영향을 미칠 수 있습니다.

반면, 컴포지션을 사용하면 클래스가 여러 동작을 동시에 수행할 수 있습니다. 이는 상속에서는 불가능한 특징입니다.
